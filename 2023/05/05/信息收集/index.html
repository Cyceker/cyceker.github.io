信息收集

信息收集的目标是使用一切合法的信息收集手段去收集有关待攻克目标的真实IP、开放端口、网络站点等信息，寻找目标的脆弱点。

信息收集范围

  1. 真实IP及其C段
  2. 开放端口
  3. 网站所有者信息（Whois）
  4. 服务类型、版本及主机系统类型
  5. Web站点及其旁站信息
  6. 敏感路径（文件）
  7. WAF识别

真实IP及其C段主机信息收集

真实 IP

拿到网站域名，收集其真实IP，有以下几种方法：

  1. ping 域名，看回显

  2. nslookup 域名

  3. 查询子域名，查询子域名IP

  4. 国内在线网站ping

    • https://ping.chinaz.com/

  5. 国外在线网站ping

    • https//asm.ca.com/en/ping.php

  6. 查DNS历史解析记录

子域名收集

  1. 在线网站查子域名
    • 站长工具
    • IPC备案网
  2. 搜索引擎：site: baidu.com
  3. 网络空间测绘引擎：
    • fofa: domain="baidu.com"
    • shodan
    • 各大安全厂商的引擎
  4. 工具枚举子域名
    • Layer子域名挖掘机
    • OneForAll
  5. 透明证书：https://crt.sh/

C段

C段为目标IP所处网段。收集C段信息是收集目标所在网段内其他存活主机的信息，是为了扩大攻击面，更广泛地寻找脆弱点。

方法：

  1. 扫描器：
    • nmap -sn IP/24
  2. 搜索引擎：
    • site: 192.168.16.x
  3. 网络空间测绘引擎：
    • fofa：ip="192.168.56.0/24"
  4. 其他工具：
    • webfinder: ip="192.168.56.0/24"

端口扫描

常用工具：

  • nmap，不多介绍，下面只列举些常用扫描方式

-sV: 扫描服务版本信息
-sA: 穿越防火墙

-p: 指定端口

-A: 全面扫描

-oN: 将扫描结果导出到普通文件中（txt）

  • masscan

  • oneforall 端口扫描

  • 其他端口扫描工具

Whois 信息

  • 站长工具
  • 企查查
  • 天眼查
  • 爱企查

Web站点信息和系统信息

Web服务器及中间件

HTTP流量检查

  1. Server 关键字
  2. 请求不存在文件报错可能显示服务器或中间件信息

Web服务识别工具

  • nmap
  • whatweb

防止显示中间件版本

防止显示服务版本信息

● apache -> /apache/httpd-default.conf -> serverTokens Prod，ServerSigniture off

● nginx ->nginx/conf/nginx.conf -> server_tokens off;

Web指纹识别

识别包括CMS（内容管理系统）、WebServer框架（Weblogic、ThinkPHP）类型等信息

判断依据有：
● 特有文件

● 特有 MD5（不是很可靠）

● 文件命名规则

● 返回头关键字

● 网页关键字

在线平台：

  • bugscaner 需要扫码登录 http://whatweb.bugscaner.com/look/

  • 数字观星 https://fp.shuziguanxing.com/#/

  • 潮汐 http://sso.tidesec.com/

  • whatweb 502 Bad Gateway

  • github查找 https://github.com/search?q=cms识别

  • 云悉 需注册 http://www.yunsee.cn/finger.html

工具：

  • 网络空间测绘引擎

    • fofa

    • 360网络空间测绘

    • 知道创宇网络空间测绘

  • 在线CMS识别

  • whatweb（kali有）

  • wappalyzer （浏览器插件）

服务器系统

  • Nmap 扫描：nmap -O

  • TTL值

    • ping 时，TTL <= 128:Windows
    • ping 时，TTL <= 64: Linux

  • 大小写敏感，改变域名大小写测试

  • 不敏感，Winodws

  • 敏感，Linux

修改TTL值

  • Windows
  • 运行->regedit-> HKEY_LOCAL_MACHINE->SYSTEM->CurrentControlSet->Services->Tcpip->DefaultTTL
  • Linux
  • CentOS：
    临时修改 echo 128 > /proc/sys/net/ipv4/ip_default_ttl，

    永久修改 vi /etc/sysc，添加 net.ipv4.ip_default_ttl=128

敏感路径（文件）

因为网站管理员配置不当，导致可以遍历网站目录，获取网站目录结构，甚至找到包含关键信息的敏感信息，如网站备份、Git信息等

WAF 识别

常见waf有：D盾、云锁、安全狗、宝塔、护卫神、华为云、阿里云、等

CSDN博客-常见WAF识别

手工识别：

  1. 云盾：响应中包含yundun、源代码有 errors.aliyun.com
  2. 安全狗：响应头包含 waf2.0、safedog等
  3. 腾讯云：waf.tencent.cloud.com，阻止响应代码405
  4. 百度云：响应头包含Yunjwsu-nginx
  5. 创云盾：URL：365cyd.com、365cyd.net

自动探测：waf00f（kali有）
